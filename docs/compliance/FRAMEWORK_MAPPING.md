# Framework Mapping

## Purpose

This document describes how policy compliance validation maps agent evaluation evidence to governance frameworks.

## Frameworks

| Framework | Validation Focus |
|---|---|
| NIST AI RMF | Govern, Map, Measure, Manage posture |
| Responsible AI | Safety, transparency, accountability, human oversight |
| SOC 2 | Security, Availability, Processing Integrity, Confidentiality, Privacy |
| HIPAA-style controls | PHI exposure, redaction, patient impact, human review |
| Internal AI policy | Approved purpose, source controls, tool boundaries, escalation |
| Model risk management | Risk tier, monitoring, review cadence, remediation |

## Evidence Outputs

Each validation record should preserve:

- run ID
- benchmark ID
- agent ID
- framework mappings
- policy outcome
- failed controls
- required controls
- escalation reason
- remediation guidance
- governance board referral
- SOC 2 trace areas

## Boundary

This mapping creates readiness evidence only.

It does not claim certification, attestation, regulatory compliance, or production operating effectiveness.
